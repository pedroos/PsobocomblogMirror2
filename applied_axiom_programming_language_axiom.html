The applied axiom & the programming language axiom

Mathematics

Pedro O S
June/2020

In ε-supremum existence and uniqueness with axiom of completeness (proof), we tried to give the proof of existence and uniqueness of the supremum in \(\mathbb{R}\). To this end, we used the axiom of completeness.

On this article, we'll elaborate on some details of the logic present in the axiom.

The axiom as defined in the proof I gave is more abstract than as would be only applied to \(\mathbb{R}\). \(c\) pre-exists in \(\mathbb{R}\) before the axiom, because of the denseness of \(\mathbb{R}\). The axiom of completeness' work generally is to give a common name \(c\) to such point, which in the reals is either \(max(A) \in A\) in case \(A\) is closed to the right, or \(min(S) \notin A\) in case \(A\) is open to the right.

These situations are, respectively, in the reals

  • \(a \leq c \leq s, \forall a \in A, \forall s \in S\), with \(A\) and \(S\) not disjoint; and
  • \(a \lt c \leq s, \forall a \in A, \forall s \in S\), with \(A\) and \(S\) disjoint, covered because \(a \leq c \leq s \Rightarrow a \leq b \leq s, \forall a, s, c\) in such ordered set.

Note the axiom in this wording is "more than sufficient" when applied to \(\mathbb{R}\) in a sense because of this, any case where, in the axiom's variables, \(c \lt b \land c \neq b\) (which would still qualify as \(c \leq b\)) never occurs, because this would imply an open-to-the-left set of upper bounds \(S\), of which there are none in \(\mathbb{R}\). In fact, it is precisely this fact that certifies \(c\) exists when \(A\) is open to the right in \(\mathbb{R}\), as for in other cases if \(S\) is open to the left there is no \(min(S)\).

So \(c \leq b\) is a tautology in the applied axiom to \(\mathbb{R}\) and the axiom could, less simply, but with a more sufficient purport, be defined

  • Let the sets \(A, B \subset \mathbb{R}\).
  • If \(a \lt b, \forall a \in A, \forall b \in B\), (or \(A\) and \(B\) are disjoint), then
    • \(\exists c \in B \mid a \lt c \leq b, \forall a \in A, \forall b \in B\text{.}\qquad\)\((1)\)
  • Else
    • If \(\exists! a \in A, b \in B \mid a = b\) (or \(A\) and \(B\) are joined by one point), then
      • \(c = a\).
    • Else
      • \(\nexists c\)
Note: To be more sufficient, we could in \((1)\) explicit \(c = min(B)\), or \(c = x \in B \mid x \leq y, \forall y \in B\). This also has the consequence that \(c\) now refers in this expression to a single point instead of many, as will be elaborated in "the programming language axiom" section below.

But what does \(\nexists c\) mean? The presence or absence of this statement mean the same, that there is no such \(c\). This harks to the nature of this axiom, which is in reality the definition of a variable. We could omit the non-existance branch, but then it would not be obvious, now taking the literality of the definition into account, in which cases the variable, which is the result of the axiom, would cease to exist. (Note that in the above definition, the non-existance branch is mutually exclusive with all other possible branches of the definition, and the sum of all possibilities yield the totality of possibilities: so there is no chance of not having a result or, more close to programming language terms, an undefined variable.)

The programming language axiom

In programming language terms, in an imperative definition, we would have the above definition, with the only difference that the non-existance of \(c\) would be substituted by essentially \(c\) existing, with a null value.

In a functional kind of programming language definition, one would define \(c\) in one step:

$$c = \text{is }a \lt b, \forall a \in A, \forall b \in B\text{? then }\\c \mid a \lt c \leq b, \forall a \in A, \forall b \in B\text{. }\color{brown}{\underline{(1b)}}\text{ if not,}\\\exists! a \in A, b \in B \mid a = b\text{? then}\\a\text{. if not, then nothing.}$$

Now, \(c\) always refers to a value, even if nothing. The only aspect in this direction left to examine is that \((1)\), \((1b)\) refer to a point defined by an inequality. We know it is not the intention of the axiom to refer to multiple points (on the contrary!), so we can try to define which point it is, so as to condense the variable value from a collection of points to a single point. (This would achieve an uniform type for the value of the variable in the programming language definition.)

But, again, back to the properties of \(\mathbb{R}\), an arbitrary proper point can't be pointed to in this set, because of its denseness. We can't count towards the point; say, to reach for the first point or any other position-based stipulation. There is no hope except for the fact that we know that the point in this case equals \(min(B)\). Let's use it in its explicit form.

$$\text{point }c = \text{is }a \lt b, \forall a \in A, \forall b \in B\text{? then }\\c \in B \mid c \leq b, \forall b \in B\text{. if not,}\\\exists! a \in A, b \in B \mid a = b\text{? then}\\a\text{. if not, then nothing.}$$
